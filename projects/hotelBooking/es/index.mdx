### Presentación
Bienvenido al sitio web de reservas de hoteles. Este proyecto, impulsado por las tecnologías de vanguardia MERN (MongoDB, Express.js, React.js, Node.js), proporciona una experiencia de usuario fluida y segura. Explora el propósito, los retos a los que nos enfrentamos y las valiosas lecciones aprendidas a lo largo del proceso de desarrollo.

### Finalidad y objetivo del proyecto

El objetivo principal del sitio web de reservas hoteleras era crear una plataforma segura y fácil de usar para las personas que buscan una experiencia vacacional excepcional. Nuestra meta era replicar la visión del diseñador y los requisitos del cliente, garantizando que los usuarios pudieran navegar y reservar hoteles fácilmente basados en impresionantes imágenes de diseño.

### Web stack y explicación

El sitio web de reservas hoteleras utilizó varias tecnologías, cada una con una función específica en el proceso de desarrollo:

- **JavaScript y TypeScript:** Estos lenguajes de programación permitieron la creación de componentes web dinámicos e interactivos.
- Node.js:** Node.js, como motor de ejecución de JavaScript del lado del servidor, facilitó el manejo de la lógica del servidor y el procesamiento de datos.
- CSS: Se utilizaron hojas de estilo en cascada para diseñar y dar estilo al sitio web, garantizando una interfaz de usuario estéticamente agradable.
- MongoDB y Mongoose:** MongoDB sirvió como base de datos NoSQL, mientras que Mongoose actuó como biblioteca de modelado de datos de objetos (ODM), simplificando las operaciones de base de datos y garantizando una integración de datos perfecta.
- **Express.js:** Este marco de aplicaciones web permitió la creación de API sólidas del lado del servidor, el enrutamiento y la implementación de middleware.
- **Bcrypt y JSON Web Token:** Estas tecnologías se emplearon para la autenticación y autorización seguras de usuarios.
- **React.js y Redux Toolkit:** React.js facilitó la creación de componentes dinámicos y reutilizables, mientras que Redux Toolkit gestionó los estados de forma eficiente.
- **Cloudinary React:** Esta tecnología permitió una integración perfecta de la carga y el alojamiento de imágenes.
- **GitHub:** GitHub sirvió como sistema de control de versiones, permitiendo una colaboración y gestión de código eficientes.

### Problemas y reflexiones

A lo largo del proceso de desarrollo, nos encontramos con retos que pusieron a prueba nuestro pensamiento crítico y nuestra capacidad para resolver problemas. En particular, gestionar los estados de Redux Toolkit y garantizar su sincronización durante la navegación del usuario resultó ser un obstáculo importante. Además, comprender e implementar los efectos secundarios de React supuso una curva de aprendizaje. La revisión exhaustiva de la documentación y la experimentación práctica nos permitieron superar estos obstáculos y lograr los resultados deseados.

### Lecciones aprendidas

El proyecto del sitio web de reservas hoteleras nos ha proporcionado enseñanzas e información valiosas que nos servirán de guía en el futuro. Aprendimos la importancia de planificar los efectos secundarios desde el inicio del proyecto, ya que mejora significativamente la experiencia de desarrollo. Además, la realización de pruebas minuciosas antes de pasar a la fase de producción o a las demostraciones para clientes demostró ser muy valiosa. De cara al futuro, será prioritario planificar cuidadosamente los efectos y las acciones de cada vista, para garantizar ciclos de desarrollo más fluidos.

### Posibles mejoras

Dado que el sitio web de reservas hoteleras es un proyecto finalizado, no se prevén más actualizaciones. Sin embargo, en retrospectiva, reconocemos posibles áreas de mejora y optimización. Una mejor planificación de la experiencia de desarrollo habría permitido una ejecución más eficiente. Al tratarse de un producto mínimo viable (MVP), habríamos priorizado los requisitos de forma más meticulosa, garantizando una implementación óptima de las funciones. La incorporación de una función de reseña para hoteles y la posibilidad de guardar métodos de pago habrían mejorado la experiencia general del usuario, aportando valor y comodidad adicionales.
